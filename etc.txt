Previously, I decided to bound the catalog, network, and datastore to the node because these three items essentially belong to each node. In the future, if I have a Proxmox cluster with multiple nodes, the template, datastore, and network might be on each node. This maintains the consistency of the health status of these three items. 
If the node goes down but the cluster remains active, the provider status will remain active, but the catalog, network, and datastore will follow the node status. 
And later in the provisioning menu, if the node goes down, the node data drop-down will not appear. Only this node has raw data that appears in provisioning but is retrieved from the DB.
