2.2	Landasan Teori
2.2.1	Cloud Computing
2.2.1.1	Definisi
Cloud Computing merupakan paradigma komputasi yang memungkinkan pengguna mengakses sumber daya teknologi informasi seperti server, penyimpanan, jaringan, dan layanan komputasi lainnya melalui jaringan internet sesuai kebutuhan. Konsep ini memberikan kemudahan bagi organisasi dalam memanfaatkan sumber daya komputasi tanpa harus memiliki dan mengelola seluruh infrastruktur fisik secara mandiri. Melalui Cloud Computing, sumber daya komputasi dapat disediakan secara fleksibel sehingga pengguna dapat menyesuaikan kapasitas layanan berdasarkan kebutuhan operasional yang dimiliki.
Nabila dan Indrawati (2025) menjelaskan bahwa Cloud Computing memungkinkan pengguna mengakses sumber daya komputasi seperti penyimpanan, pemrosesan, dan jaringan melalui internet tanpa harus membeli serta mengelola perangkat keras secara langsung. Pendekatan ini memberikan fleksibilitas yang tinggi dalam pengelolaan infrastruktur sekaligus mendukung efisiensi penggunaan sumber daya teknologi informasi.
Perkembangan Cloud Computing didorong oleh kebutuhan organisasi terhadap infrastruktur yang lebih fleksibel, mudah dikembangkan, dan mampu menyesuaikan perubahan kebutuhan layanan secara cepat. Padur (2021) menjelaskan bahwa transformasi pengelolaan infrastruktur dari pendekatan tradisional menuju platform berbasis kode dan cloud mendorong penerapan otomatisasi, skalabilitas, serta pengelolaan infrastruktur yang lebih terstruktur. Kondisi tersebut menjadikan Cloud Computing sebagai salah satu fondasi utama dalam pengelolaan layanan teknologi informasi modern.
2.2.1.2	Model Layanan Cloud Computing
Cloud Computing menyediakan beberapa model layanan yang dapat digunakan sesuai kebutuhan pengguna. Nabila dan Indrawati (2025) menjelaskan bahwa model layanan Cloud Computing terdiri dari Infrastructure as a Service (IaaS), Platform as a Service (PaaS), dan Software as a Service (SaaS). Ketiga model tersebut memiliki karakteristik dan tingkat pengelolaan yang berbeda sesuai dengan kebutuhan pengguna.
Infrastructure as a Service (IaaS) menyediakan sumber daya infrastruktur seperti server virtual, penyimpanan, dan jaringan yang dapat dikelola secara mandiri oleh pengguna. Pada model ini, pengguna memiliki kendali terhadap sistem operasi dan konfigurasi layanan yang dijalankan. Platform as a Service (PaaS) menyediakan lingkungan pengembangan aplikasi yang memungkinkan pengguna membangun dan menjalankan aplikasi tanpa harus mengelola infrastruktur dasar yang digunakan. Sementara itu, Software as a Service (SaaS) menyediakan aplikasi yang dapat digunakan secara langsung melalui jaringan internet tanpa memerlukan instalasi maupun pengelolaan sistem secara mandiri.
Penerapan model layanan tersebut memberikan fleksibilitas bagi organisasi dalam menentukan tingkat pengelolaan infrastruktur yang diperlukan. Selain itu, penggunaan layanan berbasis cloud memungkinkan organisasi melakukan penyesuaian kapasitas sumber daya secara dinamis sehingga dapat meningkatkan efisiensi operasional dan mengurangi biaya investasi perangkat keras.
2.2.1.3	Cloud Computing dalam Penelitian
Cloud Computing menjadi salah satu landasan utama dalam penelitian ini karena sistem yang dikembangkan bertujuan untuk menyediakan layanan provisioning mesin virtual secara mandiri melalui aplikasi web self-service. Konsep penyediaan sumber daya komputasi sesuai kebutuhan pengguna merupakan karakteristik yang sejalan dengan implementasi Cloud Computing, khususnya pada model Infrastructure as a Service (IaaS).
Dalam penelitian ini, pengguna dapat melakukan permintaan pembuatan mesin virtual melalui aplikasi web tanpa harus melakukan konfigurasi langsung pada infrastruktur virtualisasi. Sumber daya komputasi seperti CPU, memori, penyimpanan, dan jaringan disediakan secara otomatis melalui proses provisioning yang memanfaatkan Terraform dan Ansible. Dengan demikian, konsep Cloud Computing digunakan untuk mendukung penyediaan layanan infrastruktur yang fleksibel, terstandarisasi, dan mudah dikelola pada lingkungan Proxmox Virtual Environment sesuai kebutuhan pengguna.
2.2.2	Virtualisasi
2.2.2.1	Definisi
Virtualisasi merupakan teknologi yang memungkinkan satu perangkat keras fisik digunakan untuk menjalankan beberapa lingkungan komputasi secara bersamaan dalam bentuk mesin virtual (Virtual Machine). Teknologi ini dikembangkan untuk meningkatkan efisiensi pemanfaatan sumber daya perangkat keras dengan cara membagi sumber daya fisik seperti prosesor, memori, penyimpanan, dan jaringan ke dalam beberapa lingkungan yang saling terisolasi. Melalui virtualisasi, berbagai sistem operasi dan layanan dapat dijalankan pada satu perangkat fisik tanpa saling memengaruhi satu sama lain.
Marzuki dkk. (2023) menyatakan bahwa virtualisasi merupakan teknologi yang bertujuan untuk menghindari pemborosan sumber daya komputasi dan meningkatkan efisiensi penggunaan perangkat keras melalui pemanfaatan mesin virtual pada satu host fisik. Dengan virtualisasi, organisasi dapat mengoptimalkan penggunaan sumber daya yang tersedia sehingga kebutuhan akan perangkat keras tambahan dapat diminimalkan.
Penerapan virtualisasi dilakukan melalui perangkat lunak yang dikenal sebagai hypervisor. Hypervisor berfungsi sebagai lapisan yang menghubungkan perangkat keras fisik dengan mesin virtual sehingga memungkinkan beberapa sistem operasi berjalan secara bersamaan pada perangkat keras yang sama. Pendekatan ini menjadikan pengelolaan infrastruktur teknologi informasi lebih fleksibel dan efisien dibandingkan penggunaan server fisik secara terpisah.
2.2.2.2	Virtual Machine
Virtual Machine (VM) merupakan implementasi utama dari teknologi virtualisasi. Virtual Machine adalah representasi perangkat lunak yang meniru fungsi komputer fisik dan memiliki komponen virtual seperti CPU, memori, penyimpanan, serta antarmuka jaringan. Setiap Virtual Machine dapat menjalankan sistem operasi dan aplikasi secara independen sebagaimana komputer fisik pada umumnya.
Nabila dan Indrawati (2025) menjelaskan bahwa Virtual Machine merupakan perangkat lunak yang berfungsi seperti komputer fisik dan mampu menjalankan sistem operasi serta aplikasi secara terisolasi. Meskipun tidak menggunakan perangkat keras fisik secara langsung, Virtual Machine tetap memiliki komponen virtual yang menyerupai komputer fisik sehingga dapat digunakan untuk menjalankan berbagai kebutuhan komputasi.
Penggunaan Virtual Machine memberikan fleksibilitas bagi organisasi dalam mengelola berbagai layanan pada satu perangkat keras fisik. Selain itu, Virtual Machine juga banyak digunakan untuk kebutuhan pengembangan, pengujian, deployment aplikasi, serta penyediaan layanan infrastruktur berbasis cloud karena proses pembuatannya dapat dilakukan dengan lebih cepat dibandingkan penyediaan server fisik.
2.2.2.3	Manfaat Virtualisasi
Penerapan virtualisasi memberikan berbagai manfaat dalam pengelolaan infrastruktur teknologi informasi. Salah satu manfaat utama virtualisasi adalah peningkatan efisiensi penggunaan sumber daya perangkat keras. Dengan menjalankan beberapa Virtual Machine pada satu server fisik, organisasi dapat memaksimalkan pemanfaatan CPU, memori, penyimpanan, dan jaringan yang tersedia sehingga mengurangi pemborosan sumber daya.
Selain meningkatkan efisiensi penggunaan sumber daya, virtualisasi juga mampu mengurangi biaya operasional karena kebutuhan perangkat keras dapat diminimalkan. Marzuki dkk. (2023) menjelaskan bahwa virtualisasi memungkinkan organisasi menghemat biaya perangkat keras, konsumsi listrik, kebutuhan ruang server, serta biaya pemeliharaan infrastruktur teknologi informasi.
Nabila dan Indrawati (2025) juga menjelaskan bahwa virtualisasi memungkinkan beberapa sistem operasi berjalan secara simultan pada perangkat keras yang sama sehingga organisasi tidak perlu menyediakan perangkat fisik terpisah untuk setiap layanan yang dijalankan. Kemampuan tersebut menjadikan virtualisasi sebagai solusi yang efektif dalam meningkatkan fleksibilitas dan efisiensi pengelolaan infrastruktur.
2.2.3	Proxmox Virtual Environment
Proxmox Virtual Environment (Proxmox VE) merupakan platform virtualisasi sumber terbuka yang digunakan untuk membangun, mengelola, dan memantau lingkungan virtualisasi dalam satu sistem terpusat. Proxmox VE dibangun di atas sistem operasi Linux dan banyak digunakan dalam pengelolaan infrastruktur teknologi informasi karena mampu menyediakan layanan virtualisasi yang fleksibel, mudah dikelola, serta mendukung kebutuhan pengembangan infrastruktur modern. Platform ini memungkinkan administrator mengelola berbagai sumber daya virtual seperti mesin virtual, jaringan virtual, dan penyimpanan melalui antarmuka yang terintegrasi.
Menurut Marzuki dkk. (2023), Proxmox VE merupakan sistem operasi virtualisasi sumber terbuka yang banyak digunakan untuk mengelola lingkungan virtualisasi server. Proxmox VE menyediakan berbagai fitur yang mendukung pengelolaan mesin virtual secara terpusat sehingga memudahkan administrator dalam melakukan konfigurasi, pemantauan, dan pemeliharaan infrastruktur virtual. Selain itu, Proxmox VE mendukung penggunaan berbagai teknologi virtualisasi yang memungkinkan pemanfaatan sumber daya perangkat keras secara lebih optimal.
Dalam lingkungan virtualisasi, Proxmox VE berfungsi sebagai platform yang menghubungkan perangkat keras fisik dengan mesin virtual yang dijalankan di atasnya. Melalui platform ini, administrator dapat membuat, mengubah, menghapus, serta mengelola sumber daya yang digunakan oleh setiap mesin virtual sesuai kebutuhan. Kemampuan tersebut menjadikan Proxmox VE sebagai salah satu platform virtualisasi yang banyak digunakan dalam implementasi infrastruktur berbasis cloud dan data center karena mampu memberikan fleksibilitas dalam pengelolaan sumber daya komputasi.
Marzuki dkk. (2023) menjelaskan bahwa Proxmox VE juga mendukung implementasi virtual networking melalui Open vSwitch yang memungkinkan pengelolaan komunikasi antar mesin virtual maupun komunikasi dengan jaringan eksternal. Dukungan terhadap teknologi jaringan virtual tersebut memberikan kemudahan dalam membangun lingkungan virtualisasi yang terintegrasi dan dapat dikembangkan sesuai kebutuhan organisasi.
Beberapa fitur utama yang tersedia pada Proxmox Virtual Environment antara lain sebagai berikut:
1.	Manajemen Mesin Virtual (Virtual Machine Management)
Proxmox VE menyediakan fasilitas untuk membuat, menjalankan, menghentikan, mengubah konfigurasi, dan menghapus mesin virtual melalui antarmuka administrasi yang terpusat.
2.	Virtual Networking
Proxmox VE mendukung pengelolaan jaringan virtual yang memungkinkan komunikasi antar mesin virtual maupun konektivitas dengan jaringan eksternal melalui berbagai teknologi jaringan virtual.
3.	Manajemen Penyimpanan (Storage Management)
Proxmox VE memungkinkan pengelolaan berbagai media penyimpanan yang digunakan oleh mesin virtual sehingga kapasitas penyimpanan dapat diatur sesuai kebutuhan.
4.	Antarmuka Berbasis Web (Web-Based Management Interface)
Seluruh proses administrasi dapat dilakukan melalui antarmuka web sehingga memudahkan administrator dalam mengelola lingkungan virtualisasi tanpa harus melakukan konfigurasi secara langsung melalui terminal.
5.	Dukungan API (Application Programming Interface)
Proxmox VE menyediakan API yang memungkinkan integrasi dengan berbagai perangkat lunak otomatisasi dan orkestrasi infrastruktur sehingga proses pengelolaan mesin virtual dapat dilakukan secara otomatis.
6.	Skalabilitas Infrastruktur
Proxmox VE memungkinkan pengelolaan sumber daya komputasi yang dapat disesuaikan dengan kebutuhan sehingga mendukung pengembangan infrastruktur secara bertahap tanpa memerlukan perubahan arsitektur yang signifikan.
Berdasarkan kemampuan tersebut, Proxmox Virtual Environment menjadi salah satu platform virtualisasi yang mendukung penerapan otomatisasi infrastruktur dan konsep Infrastructure as Code. Dukungan terhadap manajemen mesin virtual, jaringan virtual, serta integrasi melalui API menjadikan Proxmox VE mampu digunakan sebagai platform utama dalam pengelolaan sumber daya komputasi secara efisien, terstruktur, dan mudah dikembangkan.
2.2.4	Infrastructure as Code (IaC)
Infrastructure as Code (IaC) merupakan pendekatan dalam pengelolaan infrastruktur teknologi informasi yang dilakukan melalui definisi dan konfigurasi berbasis kode. Pendekatan ini memungkinkan infrastruktur seperti server, jaringan, penyimpanan, dan layanan lainnya dikelola secara otomatis menggunakan berkas konfigurasi yang terstruktur. Dengan Infrastructure as Code, proses penyediaan dan pengelolaan infrastruktur tidak lagi dilakukan secara manual, melainkan dapat dijalankan secara otomatis dan konsisten melalui kode yang dapat disimpan, diperbarui, serta digunakan kembali sesuai kebutuhan.
Perkembangan Infrastructure as Code didorong oleh meningkatnya kebutuhan organisasi terhadap pengelolaan infrastruktur yang lebih cepat, konsisten, dan mudah direproduksi. Padur (2021) menjelaskan bahwa transformasi dari pendekatan berbasis skrip menuju platform berbasis kode memungkinkan organisasi mengelola infrastruktur secara lebih terstandarisasi, terdokumentasi, dan terintegrasi dengan proses otomatisasi. Pendekatan tersebut membantu organisasi mengurangi kompleksitas pengelolaan infrastruktur sekaligus meningkatkan kemampuan dalam melakukan pengembangan dan pemeliharaan sistem secara berkelanjutan.
Infrastructure as Code memungkinkan seluruh konfigurasi infrastruktur disimpan dalam bentuk kode sehingga setiap perubahan dapat dilacak, dikelola, dan diterapkan kembali dengan hasil yang konsisten. Jangam dan Muntala (2025) menyatakan bahwa Infrastructure as Code meningkatkan efisiensi, reliabilitas, dan skalabilitas pengelolaan infrastruktur karena konfigurasi tersimpan secara terstruktur dan dapat diterapkan secara otomatis pada berbagai lingkungan komputasi. Pendekatan ini juga membantu mengurangi risiko kesalahan konfigurasi yang sering terjadi pada proses pengelolaan infrastruktur secara manual.
Salah satu karakteristik utama Infrastructure as Code adalah kemampuan untuk menghasilkan konfigurasi yang konsisten pada setiap proses deployment. Infrastruktur yang telah didefinisikan dalam bentuk kode dapat diterapkan berulang kali dengan konfigurasi yang sama sehingga mengurangi terjadinya perbedaan konfigurasi antar lingkungan. Selain itu, Infrastructure as Code juga mendukung proses dokumentasi yang lebih baik karena seluruh konfigurasi tersimpan dalam bentuk berkas yang dapat diperiksa dan dikelola secara sistematis.
Penerapan Infrastructure as Code memberikan berbagai manfaat dalam pengelolaan infrastruktur teknologi informasi. Beberapa manfaat tersebut meliputi peningkatan efisiensi operasional, percepatan proses deployment, pengurangan kesalahan konfigurasi, kemudahan pemeliharaan sistem, serta peningkatan skalabilitas infrastruktur. Dengan memanfaatkan pendekatan berbasis kode, organisasi dapat melakukan penyediaan dan pengelolaan sumber daya komputasi secara lebih cepat dibandingkan metode manual yang memerlukan konfigurasi satu per satu pada setiap perangkat atau layanan yang digunakan.
2.2.5	Terraform
Terraform merupakan perangkat lunak Infrastructure as Code yang digunakan untuk melakukan provisioning dan pengelolaan infrastruktur secara otomatis melalui pendekatan deklaratif. Terraform memungkinkan pengguna mendefinisikan infrastruktur yang akan dibangun dalam bentuk kode sehingga proses penyediaan sumber daya dapat dilakukan secara konsisten dan terstandarisasi. Dengan pendekatan ini, berbagai komponen infrastruktur seperti mesin virtual, jaringan, penyimpanan, maupun layanan lainnya dapat dibuat dan dikelola secara otomatis tanpa harus melakukan konfigurasi manual pada setiap komponen yang digunakan.
Terraform bekerja dengan membaca berkas konfigurasi yang berisi definisi infrastruktur yang diinginkan. Berdasarkan konfigurasi tersebut, Terraform akan melakukan analisis terhadap kondisi infrastruktur yang tersedia dan menentukan perubahan yang perlu dilakukan untuk mencapai kondisi yang telah didefinisikan. Pendekatan ini memungkinkan proses provisioning dilakukan secara terstruktur serta memudahkan pengelolaan perubahan konfigurasi pada lingkungan infrastruktur yang terus berkembang.
Nelmiawati dkk. (2025) menjelaskan bahwa Terraform dapat digunakan untuk mengotomatisasi proses provisioning server sehingga sumber daya komputasi dapat disediakan secara lebih cepat dan konsisten dibandingkan metode manual. Penggunaan Terraform juga membantu mengurangi risiko kesalahan konfigurasi karena seluruh definisi infrastruktur tersimpan dalam bentuk kode yang dapat digunakan kembali sesuai kebutuhan.
Penelitian yang dilakukan oleh Nabila dan Indrawati (2025) menunjukkan bahwa Terraform memiliki efisiensi yang baik dalam pengelolaan cloud server melalui pendekatan Infrastructure as Code. Terraform mampu melakukan provisioning sumber daya secara otomatis sehingga proses penyediaan infrastruktur menjadi lebih efektif dan terukur. Selain itu, Terraform mendukung pengelolaan infrastruktur dalam skala besar karena konfigurasi dapat diterapkan secara berulang tanpa memerlukan konfigurasi manual pada setiap sumber daya yang dibuat.
Beberapa keunggulan Terraform antara lain mampu mengotomatisasi proses provisioning, mendukung pengelolaan infrastruktur berbasis kode, meningkatkan konsistensi konfigurasi, mempermudah dokumentasi infrastruktur, serta mendukung pengembangan infrastruktur yang bersifat skalabel. Dengan kemampuan tersebut, Terraform menjadi salah satu perangkat lunak yang banyak digunakan dalam implementasi Infrastructure as Code pada lingkungan cloud maupun virtualisasi modern.
2.2.6	Ansible
Ansible merupakan perangkat lunak otomatisasi yang digunakan untuk melakukan konfigurasi, pengelolaan, dan administrasi infrastruktur teknologi informasi secara otomatis. Ansible bekerja menggunakan pendekatan agentless, yaitu tidak memerlukan instalasi perangkat lunak tambahan pada server target karena komunikasi dilakukan melalui protokol yang telah tersedia, seperti Secure Shell (SSH). Pendekatan tersebut menjadikan Ansible lebih mudah diterapkan dan dikelola dibandingkan beberapa solusi otomatisasi lainnya yang memerlukan agen pada setiap perangkat yang dikelola.
Hariyadi dan Marzuki (2020) menjelaskan bahwa Ansible dapat digunakan untuk melakukan manajemen konfigurasi server secara otomatis sehingga berbagai proses administrasi yang sebelumnya dilakukan secara manual dapat dilakukan dengan lebih cepat dan konsisten. Penggunaan Ansible membantu mengurangi intervensi manual dalam pengelolaan server serta meningkatkan keseragaman konfigurasi pada lingkungan infrastruktur teknologi informasi.
Khumaidi (2021) juga menjelaskan bahwa Ansible mampu mengotomatisasi berbagai aktivitas pengelolaan server dan akun pengguna sehingga pekerjaan yang bersifat repetitif dapat dilakukan secara lebih efisien. Melalui pendekatan otomatisasi, administrator dapat mengelola banyak server secara bersamaan tanpa harus melakukan konfigurasi satu per satu pada setiap server yang digunakan.
Ansible menggunakan berkas konfigurasi yang disebut playbook untuk mendefinisikan tugas-tugas yang akan dijalankan pada server target. Playbook ditulis dalam format yang terstruktur sehingga mudah dibaca, dipelihara, dan dikembangkan sesuai kebutuhan. Dengan menggunakan playbook, proses konfigurasi server dapat dilakukan secara konsisten pada berbagai lingkungan yang berbeda sehingga membantu menjaga keseragaman konfigurasi sistem.
Selain digunakan untuk konfigurasi server, Ansible juga banyak dimanfaatkan dalam proses deployment aplikasi, manajemen jaringan, otomasi infrastruktur, serta implementasi server hardening. Nelmiawati dkk. (2025) menunjukkan bahwa penggunaan Ansible dalam proses hardening server mampu menghasilkan konfigurasi keamanan yang lebih konsisten dibandingkan konfigurasi manual. Kemampuan tersebut menjadikan Ansible sebagai salah satu perangkat lunak yang banyak digunakan dalam implementasi otomatisasi infrastruktur modern.
Beberapa keunggulan Ansible antara lain mudah diimplementasikan, tidak memerlukan agen pada server target, mendukung otomatisasi berbagai tugas administrasi, meningkatkan konsistensi konfigurasi, serta mampu mengurangi kesalahan yang disebabkan oleh proses konfigurasi manual. Dengan berbagai kemampuan tersebut, Ansible menjadi salah satu komponen penting dalam penerapan otomatisasi infrastruktur dan Infrastructure as Code pada lingkungan teknologi informasi modern.

Sistem yang dikembangkan dalam penelitian ini mengintegrasikan berbagai komponen yang sebelumnya masih berdiri sendiri menjadi satu platform berbasis web. Sistem menyediakan fasilitas self-service provisioning yang memungkinkan pengguna mengajukan permintaan pembuatan mesin virtual secara mandiri melalui antarmuka web. Setelah mesin virtual berhasil dibuat, proses konfigurasi dan hardening sistem dilakukan menggunakan Ansible sehingga menghasilkan lingkungan yang lebih terstandarisasi dan mudah dikelola.
Berdasarkan hasil perbandingan tersebut, dapat disimpulkan bahwa sistem yang dikembangkan tidak hanya memanfaatkan kemampuan otomatisasi yang dimiliki oleh Terraform dan Ansible, tetapi juga menambahkan fitur manajemen layanan berbasis web, pengelolaan pengguna berdasarkan peran, inventaris mesin virtual, serta mekanisme persetujuan layanan yang belum ditemukan secara terintegrasi pada perangkat maupun penelitian terdahulu yang digunakan sebagai pembanding.
